Uso de componentes de Origen no fiable Es emplear dependencias de origines desconocidos que pueden contener vulnerabilidades
Uso de componentes de vulnerables conocidos Se da por usar componentes desactualizados que pueden contener una vulnerabilidad
redireccionamiento y Reenvios no Validos Son redirects y forwards sin validar: en este caso el atacante es capaz de redirigir al cliente a sitios vulnerables El problema se debe al uso de entradas de usuario no validas para construir destinos de URL de aplicaciones
Detalle Error Esta vulnerabilidad se presenta cuando la aplicacion lanza un error un error ya se por no encontrar un path valido o un archivo existente en este caso se muestra informacion sensible en el detalle del error que un atacante puede usar para realizar un ataque

En la imagen adjunta podemos identificar que en el detalle del error se muestra el sistema operativo, la version del servidor y la version del codigo, con esta informacion el haker puede realizar un ataque explotando vulnerabiidades conocidas sobre la informacion obtenida
Divulgacion de Datos sensibles En este caso una peticion puede devolver informacion sensible que un atacante puede usar para realizar un ataque en este caso se recomienda exponer la menor cantidad de informacion posible o encriptar
Informacion de Depuracion/ Configuracion de seguridad Erronea En esta caso el atacante realiza un ataque para ver mas detalle de un error, para evitar que la aplicacion explote con esta vulnerabilidad se debe:

1. configurar con seguridad el acceso a los archivos, colocando archivos en el exterior del directorio web raiz.
2. Aplicar restricciones al sistema de archivos, o deshabilitar el listado de directorios
3. Nunca retornar el detalle de los errores.
Inyeccion de codigo Precisamente se trata que un usuario puede ingresar codigo para ver las caracteristicas del equipo, escalar privilegios o modificar data, por eso Nunca se debe confiar en las entradas de usuarios

* la funcion eval() en Javascript no es recomendado ya que tiene muchas vulnerabiidades
Desearilizacion de Datos no Fiables Serializar un dato es tomar un objeto y volverlo mas pequeño y seguro para almacenarlo y Desearilizar es lo opuesto es tomar ese formato seguro y pequeño para poderlo transmitir
Email Header Injeccion En algunas aplicaciones nos permite enviar preguntas sin conocer el destinatario en este caso el haker puede inyectar o agregar su correo en todas las consultas y de esta manera puede preparar un ataque ya sea:

1. Conocer los demas usuarios.
2. Lanzar una campaña por medio de la aplicacion victima.
3. Agregar contenido a una respuesta oficial de la pagina.
External Entity Injection (XXE) Un atacante puede agregar contenido XML para obtener informacion sensible
