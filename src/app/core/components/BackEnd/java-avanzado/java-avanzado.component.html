SAML Security Assertion Markup Language: Standar internacional para inicio de session unico Single Sing-On (SSO)

* Identity Provider (Punto de autenticación centralizado): verifica si el cliente tiene los permisos necesarios.

* Services Provider: proveé de los recursos solicitados al usuario autorizado.
Lo anterior es para que una vez autenticado pueda acceder a varias aplicaciones sin tener que logearse una y otra vez como es posible evidenciarlo en la 2da imagen

When
1. SAML Request: Cuando el SP envia un request a IDP.
2. SAML Response: Cuando el IDP le responde al SP.
3. Assertion: Declaraciones que estan dentro de la Respuest.
4. XML Signatures (Dsig) las Declaraciones estan firmadas usando firmas xml.
5. Assertion Consumer Service: dentro de la respuesta que el IDP le envia el SP es este ultimo de validar que todo este bien tanto las firmas como la estructura.
6. Attributes: Data sobre el usuario que se esta validando.
7. realy State: Estado de retransmisión, para que el IDP recuerde donde estaba un usuario antes de la autenticación de modo que si el usuario navega de forma anónima através del SP y decide activar algo que requiere autenticación lo reenviará al IDP

8. SAML Trust Entre el IDP y SP hay un certificado de firma o tambien llamado "idetificación de identidad" o "Emisor" 9.Metadata: permite la autoconfiguración entre el IDP y SP, lo que significa que en lugar de obtener manualmente un certificado de diferentes URL de punto final e información del emisor, etc con los metadatos se comparte un solo archivo de configuración xml
