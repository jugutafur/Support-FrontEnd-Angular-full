DataSetUn objeto DataSet es un contenedor, similar lógicamente a una base de datos. Es el que contendrá la totalidad de datos del conjunto sobre el que se desea trabajar. representa en cache datos en memoria
DataTableEl tipo DataTable representa, como su propio nombre indica, una tabla. Un objeto DataTable, si bien puede existir independientemente, está lógicamente contenido en un DataSet.
DataColumnLa clase DataColumn es el equivalente .NET de las columnas SQL. Los DataTable poseen varios DataColumn, como las tablas en base de datos poseen varias columnas.
DataRowLos objetos DataRow son los verdaderos contenedores de los datos. Cada DataRow se corresponde con un registro en un DataTable.

Shared En cada clase podemos definir una serie de atributos y metodos que cuando instanciamos un nuevo objeto estos pueden usar, es decir que con cada nuevo objeto que creemos vamos a tener una copia independiente de esos atributos y metodos.. Ahora con un atributo o metodo declarado como static para Java o Shared para Vb cada objeto no va a tener una copia de esa esa variable o metodo que declaremos con static o shared sino que van a compartir una sola variable o metodo para todos los objetos que creemos

y al igual que en Java en VB no es necesario instanciar un nuevo obejto para usar el atributo o metodo declarado con static o shared

Asociacion El concepto de asociacion es el tener dos clases separadas para unirlas por medio de un atributo o metodo

El la asociacion hay dos terminos o dos partes importantes.

1. la clase que inicia la asociacion.
2. la clase asociada

En resumen la clase que inicia la asociacion va a tener como atributo los atributos y metodos de la clase asociada de la forma en la cual se muestra la imagen
