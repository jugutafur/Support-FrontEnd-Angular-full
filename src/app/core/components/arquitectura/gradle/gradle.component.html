El complemento de Java Para que funcione bien las tareas de gradle se requiere inicialmente en el build.gradle El complemento de Java:

plugins {{abrellave}}
id 'java'
{{abrellave}}

El anterior bloque de codigo se agregan plugins, en este caso agrega el plugin de Java o complemento de Java agregando las tareas que se ven en la imagen.

hay tareas de:
* Tareas por procesables.
* Tareas de conjunto de fuentes.
* Tareas del ciclo de vida.

* Las tareas procesables en Gradle son tareas que realizan un trabajo real, como compilar código.

*Las tareas del ciclo de vida son tareas que no funcionan por sí mismas. Estas tareas no tienen acciones; en cambio, agrupan tareas procesables y sirven como objetivos para la compilación.
Gestion de Dependencias hay depedencias que pueden estar presentes en tiempo de:

1. compilacion (compile)
2. ejecuccion (runtime)
3. pruebas (test)

Creacion de task Una tarea tiene proiedades (ext) y comportamientos(doFirst, doLast.. dependOn, )

De la segunda forma podemos capturar parametros que vengan en el script

-D parametro del Systema
-P parametro del Proyecto

Tener en cuenta que de la segunda manera debemos establecer la variable en el archivo gradle.properties unicamente para las variables del proyecto ... ya que si son variables del Systema no es necesario.

Es importante mencionar que el bloque doFirst lo va a realizar primero que doLast pero antes que doFirst se realizara los parametros "ext"

otra forma

Propiedades Una tarea puede definir propiedades como se ve en la imagen que puede usar la misma tarea o ser usadas por otra tarea

tambien es posible definir las propiedades asi:

ext.nombre = "pepito perez"
ext.identificacion = "123456"

doFirst y doLast Es posible separar los bloques de codigo como lo muestra la imagen
dependsOn Es posible que una tarea dependa de otra, en la imagen se ven 3 formas direfentes de realizar esta accion.

OnlyIf Al igual que depensOn es una clausula que evalua una condicion ,, en este caso la tarea deplegar evalua si compilar ya se ejecuto sino es asi no imprime nada por consola.

Pero si se ejecuta primero compilar y despues ejecutar la clausula OnlyIf devuelve true y funciona desplegar.
mustRunAfter con esta linea obligamos a que si o si se ejecute la tarea B despues de A sin importar que se envie por linea de comandos la tarea B "primero",,,, siempre garantizando que se ejecute primero A y despues B
