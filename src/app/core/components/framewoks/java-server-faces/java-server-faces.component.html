JSF es una especificación o framework que hace parte de JEE que define un marco de trabajo MVC para el desarrollo de aplicaciones web basado en un modelo de componentes de interzas de usuario del lado del servidor

JSF: Java Server Faces usa JSP, una de las diferencias entre JSP y JSF es que JSF todo son formularios y JSP son link para el redireccionamiento
JSP: Java Server Pages = Es una tecnologia orientada a crear paginas web con programacion Java en el Servidor, estas estan compuetas con HTML/XML mezclado con etiquetas especiales para programar scripts de servidor en sintaxis java

ASP: Active Server Pages = Es una plataforma de desarrollo creada por Microsoft usado para el desarrollo de las tecnologias de servidor lo anterior significa que lo que desarrollemos en PHP, ASP, JSF, JSP son paginas que estan dentro del servidor y que son dinamicas es decir solo van a generar HTML para que el navegador interprete y renderize Solo con la informacion necesaria

ASP.NET: no es lo mismo ASP que ASP.NET, lo segundo es la evolucion de lo primero ademas ASP solo permite programar en Visual Basic en cambio ASP.NET permite varios lenguajes como C#

Tal y como veíamos en la sesión anterior, y definiéndolo de una forma muy simple, JSF es un framework orientado a recoger datos del usuario, pasarlos a la capa del modelo de la aplicación, realizar las acciones correspondientes en la aplicación y pintar los datos resultantes. Todo ello en un entorno web, con peticiones HTTP y páginas HTML.

Los datos se introducen y se muestran en forma de texto, y se almacenan en un formato dependiente de la aplicación. Por ejemplo, una fecha se puede representar con un formato dd-mm-aaaa mientras que su representación interna puede ser un objeto de la clase java.util.Date . Para realizar esta conversión entre el texto y el formato interno, se asocian al componente validadores y conversores . ver info Link 1

ManagedBean //Controllers * Clases Java de respaldo para las operaciones a realizar en las paginas web, que soportan la logica del negocio
* Requieren ser invocados con la expresion EL = {{abrellave}}#bean.atributo{{cierrallave}} * Esta clase debe estar mappeada en un archivo de configuracion "faces-config.xml"(JSF 1) o atraves de anotaciones (JSF 2+)
* Dicho lo anterior este archivo de configuracion contiene la caracterizacion del comportamiento, la ubicacion y los datos de la clase los tag:

managed-bean : Es un apodo o alias a la clase que vamos a mapear

managed-class : Es la ruta en donde esta ubicada la clase

managed-scope : Describimos el ambito de la clase, ademas el comportamiento,

navigation-rule : Son las reglas de navegacion debemos indicar que salida dara la pagina y despues indicar hacia donde se debe dirigir
