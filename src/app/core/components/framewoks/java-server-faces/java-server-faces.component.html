Scope El alcance

@RequestScoped = en cada peticion que haga el usuario hacia el controlador se va a estar creando un nuevo objeto (por ejemplo cuando haces una busqueda donde el dato a buscar sea diferente en cada momento)

@ViewScoped = Este vive a nivel de pagina es decir cuando el usuario da refresh a la pagina o cambia de pagina el bean vuelve a generarse

@SessionScoped = va a estar a nivel de session, es decir no se va a destruir este bean hasta que se destruya la sesiion del usuario (mantener las credenciales del usuario (usuario y contraseña) )

@ApplicationScoped = Esta activo en toda la aplicacion, por lo general se emplea para objetos que no van a cambiar como una lista de paises



@ViewScope Que el alcance del Bean a a ser a nivel de vista en el momento que el usuario de refresh a la pagina los datos del Bean desaparecen: Se Destruyen el ManagedBean y se crea uno nuevo
